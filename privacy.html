Privacy Policy

How we handle your data

Overview

Kincode+ is designed with privacy as a core principle. This Privacy Policy explains how we handle information when you use our desktop application.

Last Updated: February 2026

Local-First Architecture

Kincode+ operates on a local-first architecture. This means:

  • Your code never leaves your local machine unless explicitly exported by you
  • Your API keys are stored securely on your device only
  • No automatic cloud synchronization occurs
  • All project data remains on your local filesystem

Data We Do Not Collect

We do not collect, store, or transmit:

  • Your source code or project files
  • Your API keys or credentials
  • Your conversation history with AI models
  • Usage analytics or telemetry
  • Personal identification information

AI Model Interactions

When you use AI features:

  • Code snippets are sent directly to your chosen AI provider (OpenAI, Anthropic, etc.)
  • These interactions follow the privacy policies of the respective AI providers
  • We do not intercept, store, or process these communications
  • You are responsible for reviewing your chosen AI provider's privacy policy

GitHub Integration

When using GitHub features:

  • Authentication is handled via OAuth
  • Your GitHub token is stored locally on your device
  • Repository operations are performed directly between your device and GitHub

Updates

The application may check for updates. This process:

  • Only retrieves version information from our servers
  • Does not send any personal data or usage information
  • Can be disabled in application settings

Contact

For privacy-related questions, please open an issue on our GitHub repository.
